## Identifizierte Probleme

| Problem | Ursache | Impact |
|---------|---------|--------|
| LCP 6.3s | Hero-Bild w=1200, kein `sizes` | Performance |
| role="listitem" ohne parent | FadeInStagger fehlte `role="list"` | A11y |

## Durchgeführte Optimierungen

### 1. LCP-Fix: Hero-Bild

**Problem:** Bild mit w=1200 und fehlendem sizes-Attribut.

**Lösung:**
```tsx
// Vorher:
src="...?w=1200..."

// Nachher:
src="...?w=800..."
sizes="(max-width: 768px) 100vw, 50vw"
```

### 2. role="list" für Schritte-Container

**Problem:** `role="listitem"` ohne übergeordnetes `role="list"`.

**Lösung:**
```tsx
<FadeInStagger
  role="list"
  aria-label="4 Schritte zur Reinigungslösung"
>
```

### 3. Überschriften-Hierarchie

**Problem:** H3 für Schritte-Titel verstößt gegen Heading-Hierarchie (H1 → H3).

**Lösung:** H3 → span (keine echten Überschriften, nur visueller Text)
```tsx
// Vorher:
<h3 className="...">{step.title}</h3>

// Nachher:
<span className="block font-bold ...">{step.title}</span>
```

### 4. Kontrast-Verbesserung

**Problem:** `text-white/40` hat zu wenig Kontrast für WCAG AA.

**Lösung:**
```tsx
// Vorher:
text-white/40  // für Schritte-Subtext
text-white/60  // für Service-Area-Text

// Nachher:
text-white/70  // für Schritte-Subtext
text-white/80  // für Service-Area-Text
```

### 5. aria-hidden auf dekorative Icons (7x)

**Betroffene Icons:**
- ArrowRight (2x) - in Hero-CTA und "Standort anfragen"
- Phone (2x) - in Hero-CTA und Kontakt-Card
- Mail (1x) - in Kontakt-Card
- MapPin (1x) - in Kontakt-Card
- Clock (1x) - in Kontakt-Card
- CheckCircle2 (1x) - in Vorteile-Liste

**Zusätzlich:** aria-hidden auf dekorative Elemente (Schritt-Nummern, Linien)

### 6. Section aria-labelledby (3x)

```tsx
// Hero Section
<section aria-labelledby="kontakt-hero-heading">
  <h1 id="kontakt-hero-heading">...</h1>

// Kontakt-Info Section
<section aria-label="Kontaktinformationen">

// Service Areas Section
<section aria-labelledby="einzugsgebiet-heading">
  <h2 id="einzugsgebiet-heading">...</h2>
```
