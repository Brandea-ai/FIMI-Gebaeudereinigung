# Report R0013 - Leistungen-Seite Accessibility

---

## Ausgangslage

Die Leistungen-Seite hatte potenzielle Accessibility-Probleme:
- Sections ohne aria-labelledby
- Suchfeld ohne aria-label
- Filter-Buttons ohne aria-pressed State
- Suchergebnisse ohne aria-live Region

---

## Durchgefuehrte Optimierungen

### 1. Section aria-labelledby (4x)

```tsx
// Vorher:
<section ref={heroRef} className="...">

// Nachher:
<section ref={heroRef} className="..." aria-labelledby="leistungen-hero-title">
  <h1 id="leistungen-hero-title">...</h1>
```

**Betroffene Sections:**
- Hero Section (`leistungen-hero-title`)
- Leistungen Grid (`leistungen-grid-title`)
- Warum FIMI (`warum-fimi-title`)
- CTA Section (`leistungen-cta-title`)

### 2. Suchfeld Accessibility

```tsx
// Vorher:
<div className="relative flex-1 max-w-md">
  <Search size={20} className="..." />
  <input type="text" placeholder="..." />

// Nachher:
<div className="relative flex-1 max-w-md" role="search">
  <Search size={20} className="..." aria-hidden="true" />
  <input type="text" placeholder="..." aria-label="Leistungen durchsuchen" />
```

### 3. Suche-Zuruecksetzen Button

```tsx
// Nachher:
<button
  onClick={() => setSearchQuery('')}
  aria-label="Suche zurÃ¼cksetzen"
>
  <X size={18} aria-hidden="true" />
</button>
```

### 4. Filter-Buttons mit aria-pressed

```tsx
// Vorher:
<div className="flex items-center gap-1.5 sm:gap-2">
  <button onClick={...}>Alle</button>

// Nachher:
<div className="flex items-center gap-1.5 sm:gap-2" role="group" aria-label="Leistungen nach Kategorie filtern">
  <button onClick={...} aria-pressed={activeFilter === 'alle' && !searchQuery}>
    Alle
  </button>
```

### 5. Suchergebnisse Live-Region

```tsx
// Vorher:
<div className="pb-4 -mt-2">
  <p>X Ergebnisse...</p>
</div>

// Nachher:
<div className="pb-4 -mt-2" aria-live="polite" aria-atomic="true">
  <p>X Ergebnisse...</p>
</div>
```
